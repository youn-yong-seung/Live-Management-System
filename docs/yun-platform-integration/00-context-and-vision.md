# 00. 배경 / 비전 / 사용자 원문

## 핵심 비전 (사용자 원문 — 2026-05-11)

> 우리는 윤플랫폼으로 사람들을 모아서 AI에 대한 모든걸 다 배울 수 있고 그리고 나중에는 여기서 계약도 이루어지고 우리는 플랫폼화가 돼서 강사들도 여기에 등록시켜놓고 콘텐츠를 만들도록 유도하면서 양질의 정보들이 모이는 곳으로 만들거야.
>
> 그러기 위해서 유저 유입을 시키는게 첫 관문일 것 같은데 우리가 매주 라이브를 진행하면서 100명 200명씩 고객 DB가 쌓이는데 그게 윤플랫폼으로 이어지면 좋겠다고 생각했어. 라이브에서 주는 무료 자료도 다 여기 있다고 데려오면서, 우리 플랫폼에서 다양한 자료들 다 받을 수 있고 사람들과 이야기도 나눌 수 있다는 장점으로 사람들을 모으고 싶은게 주 목적이야.
>
> 그래서 윤플랫폼과 라이브 시스템을 따로 만들게 아니라 차라리 통합시켜서 시너지를 내고자 했고 사이즈를 보아하니 내생각엔 라이브 시스템을 윤플랫폼에 이식시키는게 아니라 반대로 윤플랫폼을 라이브시스템에 이식시키는게 낫겠다고 판단했음.
>
> 다만 디자인과 UI의 경우 라이브시스템보다 윤플랫폼의 디자인이 더 예쁘다고 생각돼서 디자인은 나중에 윤플랫폼식 디자인으로 교체하고 싶음

## 비전 풀어쓰기

**최종 모습**
- 단일 플랫폼: 윤자동 클래스 (도메인 미정, Live-Management-System 레포 기반)
- 사용자가 AI 관련 모든 것을 배우는 허브 — 라이브, 다시보기, 자료, 커뮤니티
- 강사 등록 / 콘텐츠 양산 / B2B 계약 / 결제까지 일어나는 곳
- yun-platform과 Live는 별도 운영하지 않음

**현재 문제 (해결해야 할 것)**
- 매주 라이브에서 100-200명의 신청자 DB가 쌓이는데 (Live의 `registrations` 테이블) 이 사람들이 yun-platform 회원으로 안 이어짐
- 라이브에서 주는 무료 자료가 외부(노션/yunjadong.com/카톡방)에 흩어져 있어서 회수 동선이 없음
- 두 시스템이 완전 별개라 시너지 0

**해결 핵심 = 유입 깔때기**
1. 라이브 신청 폼 → 신청 즉시 매직링크 발송 → 클릭 한 번 = 플랫폼 회원
2. 자료실 다운로드 → 로그인 게이트 → 회원이면 바로, 비회원이면 가입 유도
3. 카톡 대기방 + 플랫폼 회원가입 = 이중 유입 동선
